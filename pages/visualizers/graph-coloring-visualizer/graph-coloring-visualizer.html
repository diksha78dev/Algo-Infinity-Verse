🎨 Greedy vs Exact — The Real Tradeoff

⚡

Greedy Is Order-Dependent

Greedy assigns each vertex the lowest-numbered color not used by its already-colored neighbors. The SAME graph can need different numbers of colors depending purely on which order vertices are processed in — never more than Δ+1 colors (Δ = max degree), but often more than necessary.

🔙

Backtracking Finds the Truth

Backtracking tries the minimum plausible color count first, exhaustively searching all valid colorings. When a partial coloring gets stuck, it backtracks and tries a different color — guaranteed to find the true chromatic number, at exponential worst-case cost.

📅

Exam Scheduling

Courses sharing at least one student become an edge — colors become exam time slots. Minimum colors = minimum exam slots needed with zero student conflicts.

🖥️

Register Allocation

Variables alive at the same time become an edge (they can't share a CPU register). Colors become physical registers — compilers use graph coloring heuristics to minimize register spills to slow memory.
